# Сценарий заполнения и отправки формы

  1. Открыть страницу программы обучения "Тестировщик ПО" https://netology.ru/programs/qa
  2. Нажать кнопку "Записаться"
  3. Вводим валидные данные полей:
* Имя: Павел/Pavel
* Телефон: +79112223344
* Е-mail: pavel.kozlov@mail.ru

# positive path:
1. вводим валидное значение в поле-Имя
2. вводим валидное значение в поле-Телефон
3. вводим валидное значение в поле-Е-mail
4. нажимаем кнопку "записаться"
* Ожидаемый результат: открылась страница оплаты

# negative path
# Вариант 1
1. поле "имя" не введено
2. вводим валидное значение в поле-Телефон
3. водим валидное значение в поле-Е-mail
4. нажимаем кнопку "записаться"
* Ожидаемый результат: сообщение пользователю "Обязательное поле для заполнения" в поле Имя

# Вариант 2
1. вводим имя из различных символов/цифр (543?*5)
2. вводим валидное значение в поле-Телефон
3. водим валидное значение в поле-Е-mail
4. нажимаем кнопку "записаться"
* Ожидаемый результат: сообщение пользователю "Поле должно состоять из букв" в поле Имя

# Вариант 3
1. вводим валидное значение в поле-Имя
2. номер телефона не заполняем (-)
3. водим валидное значение в поле-Е-mail
4. нажимаем кнопку "записаться"
* Ожидаемый результат: сообщение пользователю "Обязательное поле для запонения" в поле Tелефон

# Вариант 4

#  Перечень используемых инструментов с обоснованием выбора.
* создание репозитоия для командной разработки и хранения проекта на GitHub https://github.com/
* IntelliJ IDEA: среда разработки, в которой пишем код https://www.jetbrains.com/ru-ru/idea/
* автоматическая сборка проекта Gradle https://1c-user.info/90-sozdanie-prostejshego-proekta-v-intellij-idea-c-gradle
* Docker (чтобы работало окружение тестирования) https://www.docker.com/
* TestContainers (чтобы при тестировании автоматически запускались необходимые контейнеры Docker при запуске тестов)
* Selenide, Selenoid (чтобы писать код, который используется именно под тестирование веб-приложения) https://ru.selenide.org/quick-start.html
* JUnit (для тестирования отдельных участков кода, например, методов или классов)
* Facker (для генерации данных)
* Allure. Фреймворк, предназначенный для создания подробных отчетов о выполнении тестов https://github.com/allure-framework/allure-maven
